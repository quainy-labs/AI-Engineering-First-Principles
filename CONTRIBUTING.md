# Contributing

Thank you for helping improve **AI Engineering from First Principles**.

This project is open for thoughtful contributions that make the book more useful, accurate, practical, and durable for AI engineering learners.

## Contribution Principles

Contributions should follow the book's core standard:

- start from real work, not AI hype;
- teach first principles before tools;
- preserve dependency order;
- prefer useful build artifacts over abstract explanation;
- include failure modes, evaluation, and tradeoffs;
- avoid vendor-specific tutorials unless they teach a durable pattern;
- avoid adding missing topics as appendices when they belong in the main learning sequence.

## Good Contributions

Good contributions include:

- clearer explanations;
- stronger examples;
- better project specs;
- missing failure modes;
- better evaluation criteria;
- corrections to inaccurate technical claims;
- diagrams or tables that clarify system design;
- improvements to chapter order when dependency requires it.

## Contributions We May Decline

We may decline contributions that:

- add hype without system design value;
- add long theory sections that do not improve builder judgment;
- promote a specific vendor, framework, or tool without a first-principles reason;
- add copyrighted material copied from books, courses, docs, blogs, or proprietary material;
- include confidential company data, customer data, secrets, or private Quainy context;
- use Quainy branding in a way that implies endorsement, partnership, or official status.

## How to Contribute

1. Open an issue for major structural changes.
2. Keep pull requests focused.
3. Explain why the change improves the reader's ability to build reliable AI systems.
4. Update links and project references if chapter order changes.
5. Run a quick link/stale-reference check before submitting.

For chapter edits, include:

```text
Problem fixed:
Files changed:
Reader benefit:
Any tradeoff:
```

By submitting a pull request, issue comment, patch, documentation edit, code snippet, diagram, or other contribution, you agree that your contribution is submitted under the **Apache License, Version 2.0**.

You also represent that:

- you have the right to submit the contribution;
- the contribution is your original work or is provided under a compatible license;
- the contribution does not include confidential, proprietary, or restricted material;
- the contribution does not knowingly infringe third-party rights.

Unless you clearly mark a submission as **"Not a Contribution"**, it may be incorporated into this project under the Apache License, Version 2.0.

## Developer Certificate of Origin

Contributors should use signed-off commits:

```text

```

This certifies that you have the right to submit the work under this project's license.

The code and content are licensed under Apache-2.0. The Quainy brand is not.

- Quainy name as your product, company, course, or community name;

Allowed:

## Forks and Derivative Works

You may fork and adapt the project under Apache-2.0, but derivative works must not imply they are official Quainy releases unless Quainy Labs has approved that in writing.

If you publish a modified version, make the modification status clear.

Recommended wording:

```text
This is an unofficial adaptation of AI Engineering from First Principles by Quainy Labs.
```

## AI-Generated Contributions

AI-assisted contributions are allowed if you review them carefully.

You are responsible for ensuring AI-assisted work is:

- accurate;
- original or license-compatible;
- free from confidential data;
- consistent with the book's structure and principles.

Do not paste generated content that imitates proprietary books, courses, docs, or private company material.

## Maintainer Rights

Quainy Labs maintainers may edit, decline, close, reorganize, or remove contributions to protect project quality, reader trust, legal safety, and brand integrity.

Maintainers may also change contribution requirements over time as the project matures.
