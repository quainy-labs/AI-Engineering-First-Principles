# Projects

These are build projects, not chapter exercises.

Each project is a concrete system that demonstrates how AI engineering concepts become product architecture. Learners should build the system, run it on sample or real data, evaluate behavior, and explain tradeoffs.

## Project Path

1. [Support Workflow Intelligence Console](01-workflow-teardown.md)
2. [Knowledge Base Search Engine](02-knowledge-base.md)
3. [Structured Intake Extractor](03-structured-extraction-system.md)
4. [Grounded Research Assistant](04-grounded-research-assistant.md)
5. [Operations Action Assistant](05-operations-assistant.md)
6. [Production AI Control Center](06-production-hardening.md)

## What Makes These Real Projects

Each project includes:

- concrete user;
- product goal;
- sample data;
- system architecture;
- core components;
- UI or CLI behavior;
- data model;
- evaluation plan;
- acceptance tests;
- extension path.

## Rule

Do not stop at documents.

Every project should produce a runnable or inspectable system:

- CLI;
- local web app;
- API service;
- notebook with reproducible pipeline;
- or small full-stack prototype.

The capstone remains learner-chosen. These six projects are guided builds that show system patterns before capstone.
